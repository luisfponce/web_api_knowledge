```dbml
Table user {
  id int [pk, increment]
  username varchar(50) [not null]
  name varchar(100) [not null]
  last_name varchar(100) [not null]
  phone bigint [null, unique]
  email varchar(100) [not null]
  hashed_password varchar(255) [not null]

  Note: 'Mapped from SQLModel User entity'

    name
    last_name
    phone [unique]

Table prompts {
  id int [pk, increment]
  user_id int [not null, ref: > user.id]
  model_name varchar(30) [not null]
  prompt_text longtext [not null]
  category varchar(30) [not null]
  rate varchar(30) [not null]

  Note: 'Mapped from SQLModel Prompts entity'
}
```

## Relationship cardinality

- One-to-many from [`prompts: List["Prompts"] = Relationship(...)`](../webapi/models/user.py:17) to [`user: Optional["User"] = Relationship(...)`](../webapi/models/prompts.py:15)
- Foreign key is defined in [`user_id: int = Field(foreign_key="user.id", nullable=False)`](../webapi/models/prompts.py:9)
- Back-population is configured via [`back_populates="user"`](../webapi/models/user.py:19) and [`back_populates="prompts"`](../webapi/models/prompts.py:17)

## Current model notes

- `phone` is optional and unique in [`phone: Optional[int] = Field(default=None, sa_type=BIGINT, index=True, unique=True)`](../webapi/models/user.py:13).
- User searchable fields use indexes in [`username`](../webapi/models/user.py:10), [`name`](../webapi/models/user.py:11), and [`last_name`](../webapi/models/user.py:12).
- Prompt ownership is mandatory through [`user_id`](../webapi/models/prompts.py:9), enforcing that every prompt belongs to a user.
- The ORM class name is pluralized as [`Prompts`](../webapi/models/prompts.py:7), which maps to the `prompts` table name generated by SQLModel.
